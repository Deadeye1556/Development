# UX Reviewer — Review Protocol

## Purpose

Consolidate `learning.md` entries into `role.md` improvements. Run only when triggered. After a review cycle, `learning.md` resets and entries are archived.

---

## Trigger Criteria

- 10+ entries in `learning.md`
- Manual trigger by PM or COO
- After two consecutive audits with routing corrections on the same issue type

---

## Constitutional Constraints — Non-Negotiable

1. **Business pillars are immutable** — Board authority only. No review cycle changes product direction or brand positioning in `role.md`.
2. **No degradation** — every change to `role.md` must improve clarity or catch a real gap. No changes that weaken the audit standard.
3. **Distill, do not append** — `role.md` must be the same length or shorter after a review cycle. Compress existing content if new entries require space.
4. **Conflicts escalate** — if a learning entry conflicts with a `role.md` standard, escalate to PM before updating. Do not resolve the conflict yourself.

---

## Process

1. **Audit `learning.md`** — read every entry since the last reset
2. **Categorize** — group by: routing corrections / missed findings / brand patterns / flow gaps / PM corrections
3. **Update `role.md`** — replace or refine existing sections only; no net new length unless something critical is missing from the dispatch table or flow checklist
4. **Quality check** — confirm `role.md` is under 900 words, dispatch table is complete, constitutional constraints are intact
5. **Archive** — copy current `learning.md` to `archive/learning-[YYYY-MM].md`
6. **Reset** — restore `learning.md` to blank template with category headers only

---

## What Is Eligible for role.md Updates

- New issue types added to the dispatch routing table (if recurring and unambiguous)
- New flows added to the standard checklist (when a milestone ships permanent features)
- New banned language added to brand voice violations
- Clarifications to ambiguous dispatch cases

## What Is Not Eligible

- One-off findings that haven't recurred
- Corrections that were specific to a single build or milestone
- Anything that would require a product scope decision
